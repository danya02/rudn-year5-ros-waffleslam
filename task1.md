Тема: Введение в ROS 2 и симуляцию TurtleBot 3 в Gazebo

Цель работы:
Познакомиться с основами Robot Operating System (ROS 2), научиться запускать симуляцию мобильного робота TurtleBot 3 в среде Gazebo, изучить базовые инструменты взаимодействия с ROS (топики, узлы, сообщения) и получить первичные навыки визуализации данных в RViz.

Оборудование и программное обеспечение:
    • Персональный компьютер с ОС Ubuntu 22.04 LTS;
    • Установленный ROS 2 Humble (или более новая версия);
    • Установленные пакеты для TurtleBot 3: turtlebot3, turtlebot3_simulations, turtlebot3_msgs, gazebo и необходимые зависимости;
    • Утилиты командной строки ROS 2: ros2, rviz2, gazebo.
Теоретическое введение
Что такое ROS?
ROS (Robot Operating System) - это гибкая распределённая система для разработки программного обеспечения роботов. Она предоставляет стандартные сервисы операционной системы (аппаратную абстракцию, низкоуровневый контроль устройств, передачу сообщений между процессами, управление пакетами). ROS 2 является новой версией, использующей DDS (Data Distribution Service) для коммуникации, что обеспечивает лучшую надёжность и подходит для промышленных применений.
Ключевые понятия ROS 2
    • Узел (Node) - исполняемый модуль, который выполняет определённую задачу (например, управление двигателями, обработка изображений с камеры). Узлы общаются друг с другом посредством сообщений.
    • Топик (Topic) - именованный канал для обмена сообщениями. Узлы могут публиковать (publish) сообщения в топик или подписываться (subscribe) на них. Топики обеспечивают асинхронную однонаправленную связь «многие ко многим».
    • Сообщение (Message) - структура данных, определяющая тип передаваемой информации (например, Twist для команд скорости, LaserScan для данных лидара).
    • Сервис (Service) - механизм синхронного взаимодействия «запрос-ответ» между узлами.
    • Пакет (Package) - основная единица организации ПО в ROS. Содержит узлы, конфигурационные файлы, launch-файлы и т.д.

TurtleBot 3 и Gazebo
TurtleBot 3 - популярная образовательная мобильная робототехническая платформа. В данной работе мы будем использовать её цифрового двойника в симуляторе Gazebo. Gazebo позволяет моделировать физику, сенсоры (лазерный дальномер, одометрию) и взаимодействие с окружением.
Ход работы
1.Установка пакетов:
    1. Обновите список пакетов:
sudo apt update
    2. Установите ROS 2:
sudo apt install ros-humble-desktop
    3. Установите TurtleBot 3 и пакеты для симуляции:
sudo apt install ros-humble-turtlebot3*
    4. Установите Gazebo:
sudo apt install gazebo11
    5. Настройте окружение:
source /opt/ros/humble/setup.bash
2. Запуск симуляции TurtleBot 3 в Gazebo
    1. Откройте терминал и запустите ROS 2 Master:

ros2 daemon start
    2. Запустите Gazebo с моделью TurtleBot 3 (выберите одну из моделей: waffle, burger, waffle_pi):

export TURTLEBOT3_MODEL=waffle
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py

    3. Убедитесь, что в окне Gazebo появился робот и окружение.
3. Изучение топологии ROS 2
    1. В новом терминале выполните:
ros2 node list
Вы увидите список активных узлов, включая gazebo и robot_state_publisher.
    2. Посмотрите список топиков:
ros2 topic list

Найдите топик для управления движением робота (обычно это /cmd_vel).
    3. Изучите тип топика:
ros2 topic info /cmd_vel

Убедитесь, что тип сообщения — geometry_msgs/msg/Twist.
    4. Просмотрите структуру сообщения:

ros2 interface show geometry_msgs/msg/Twist

Обратите внимание на поля linear.x (движение вперёд/назад) и angular.z (поворот).
4. Управление роботом через командную строку
    1. Отправьте команду на движение вперёд:
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.2, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

Робот должен начать движение вперёд.
    2. Отправьте команду на поворот:
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.5}}"

Робот начнёт поворачиваться против часовой стрелки.
    3. Остановите робота:
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}" -r 1

5. Визуализация данных в RViz 2
    1. Запустите RViz 2:

ros2 run rviz2 rviz2

    2. Добавьте отображение робота:
    • нажмите Add → RobotModel;
    • в поле Description Topic укажите /robot_description.
    3. Добавьте отображение камеры (если есть):
    • Add → Image;
    • укажите топик камеры (например, /camera/image_raw).
    4. Добавьте отображение лидара (если есть):
    • Add → LaserScan;
    • укажите топик лидара (например, /scan).
    5. Изучите данные сенсоров в 3D‑окне RViz.

6. Задания для самостоятельной работы
    1. Создайте простой скрипт на Python, который заставит робота двигаться по квадрату. Используйте цикл и отправку команд в топик /cmd_vel.
    2. Напишите скрипт, который заставит робота объезжать препятствия (используйте данные лидара из топика /scan для обнаружения препятствий).
    3. Измените модель окружения в Gazebo: отредактируйте файл turtlebot3_world.sdf, добавив стены или препятствия.
    4. Запустите два робота в одном окружении: измените параметры запуска, чтобы создать два отдельных робота с разными именами узлов.
